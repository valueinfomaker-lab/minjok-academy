# 05. 자산 요청 체크리스트

> 학원 측이 추후 제공해야 할 사진·정보·콘텐츠 리스트.
> 1차 HTML 구현은 더미값/Unsplash 사진으로 진행하고, 본 리스트 자산이 확보되면 순차 교체한다.

---

## 1. 텍스트 정보 (즉시 확보 필요)

### 학원 기본 정보

- [ ] **정확한 학원명** (간판/공식명 — "민족 어학원" vs "민족영어학원")
- [ ] **사업자 등록번호**
- [ ] **대표자 이름** (원장)
- [ ] **정확한 주소** (도로명 + 지번 + 건물명 + 층)
- [ ] **대표 전화번호**
- [ ] **대표 이메일**
- [ ] **카카오톡 채널 ID** (보유 시)
- [ ] **인스타그램 / 블로그 / 유튜브 URL** (보유 시)
- [ ] **운영 시간**
  - 유치부 (현재 안내: 09:00 ~ 15:00 — 정확한 요일 확정 필요)
  - 초·중등부 (요일별 시간대)
- [ ] **휴원일 / 방학 일정**

### 원장 인사말

- [ ] **원장 이름**
- [ ] **원장 약력** (학력, 경력 — 강사진 페이지에도 활용)
- [ ] **원장 인사말 전문** (현재 02-content-strategy.md 의 인사말은 사용자 제공 문장 + 가공한 초안. 실제 원장 톤으로 교체 필요)
- [ ] **원장 사진** (정장/세미캐주얼, 사선 또는 정면, 600×800px 권장)

### 16년 연혁

연도별 주요 이벤트 확인 필요 (현재 더미 기반):

- [ ] **2010** — 정확한 개원일 (월/일)
- [ ] **2010~2018** 사이의 확장 이벤트들 (2013 통합 확장은 더미)
- [ ] **2019년 6층 유치관 개관** — 정확한 일자
- [ ] **누적 졸업생/학생 수** — 정확한 숫자
- [ ] **수상 / 인증** 내역 (있다면)

### 강사진

각 강사별:
- [ ] 이름 (또는 한글 성 + 영문 이름)
- [ ] 사진 (정장, 400×400px, 자연광)
- [ ] 학력 (명문대 출신 강사 강조 가능)
- [ ] 경력
- [ ] 담당 과목 / 학년
- [ ] 한 줄 소개

> 예: 김민지 선생님 / 서울대학교 영어교육학과 / 10년 경력 / 초등 Grammar·Reading / "정확한 문법으로 흔들리지 않는 영어 실력을 키웁니다"

### 프로그램 상세

#### 유치부
- [ ] 정확한 일과표 (현재 와이어프레임의 시간표는 추정)
- [ ] 사용 교재 목록
- [ ] 분반 기준 (연령별? 레벨별?)
- [ ] 정원 / 현재 분반 수
- [ ] 월 학원비 (공개 여부 결정)

#### 초등부
- [ ] 학년별 시간표
- [ ] 사용 교재 (학년별)
- [ ] 분반 / 레벨 시스템
- [ ] 진단 평가 절차
- [ ] 자체 워크북 보유 여부

#### 중·고등부
- [ ] 학년별 커리큘럼
- [ ] 내신 대비 학교 리스트 (인근 중고교)
- [ ] **특목고 진학 실적** — 가능한 범위 내에서 (수치/학교명)
- [ ] 시험 대비 일정 (TOEFL, 텝스, 모의고사 등)

## 2. 사진 자산

### 우선순위 P0 — 메인 페이지에 즉시 필요

- [ ] **Hero 배경 사진 1장** (메인 페이지 최상단)
  - 권장: 유치관에서 원어민 선생님과 아이들이 활동하는 자연스러운 모습
  - 가로형, 2880×1620 이상
- [ ] **연령별 프로그램 카드 사진 3장**
  - 유치부 / 초등부 / 중·고등부 각 1장
  - 4:3 비율, 800×600 이상

### 우선순위 P1 — 유치부 페이지

- [ ] **6층 유치관 시설 사진** (최소 6장)
  - 메인 교실
  - 라운지 / 휴식 공간
  - Kids Cafe
  - 화장실 (선택)
  - 식사 공간
  - 외관 또는 입구
- [ ] **활동 사진** (각 2~3장)
  - English Craft
  - English Cooking
  - Gym (In-line Skate)
  - Kids Cafe 활용 모습

### 우선순위 P3 — 보조 콘텐츠 (필요 시 추가)

- [ ] **점심 사진** (활동 메뉴 중 하나 또는 일상 콘텐츠로 활용)
  - 식판 위 반찬 + 밥 + 국 (1~2장 정도)
  - 조리 중인 모습 (있다면)

### 우선순위 P1 — 초·중등부 페이지

- [ ] **5층 초중등관 시설 사진** (4~6장)
  - 강의실
  - 자습실 (있다면)
  - 상담실
  - 외관 또는 입구
- [ ] **수업 모습** (원어민 / 한국인 강사 각각)

### 우선순위 P2 — 갤러리 / 학원소개

- [ ] **연간 행사 사진** (할로윈, 크리스마스, 졸업식 등)
- [ ] **강사진 단체 사진**
- [ ] **학부모 만족도 행사 / 학예회**
- [ ] **로고** (벡터 SVG 또는 고해상도 PNG, 투명 배경)

> **로고가 없는 경우**: 1차 HTML 구현 시 텍스트 로고 ("민족 어학원" / Pretendard ExtraBold + 골드 라인 underbar) 로 진행. 추후 디자이너에게 의뢰 권장.

### 촬영 가이드 (학원 자체 촬영 시)

- 가능하면 **자연광** (창 측에서 조명)
- 핸드폰 촬영 시 가로/세로 통일, 가로 촬영 권장
- 인물 사진은 **반드시 학부모 동의서** 수령 후 사용
- 아이들 얼굴이 정면으로 나오는 사진은 동의 까다로움 → **뒷모습, 측면, 손/활동 중심** 사진을 더 많이 확보 권장
- 음식·소품 등 정물 사진은 **위에서 내려다보는 각도**로

## 3. 후기 / 평판

- [ ] **학부모 후기** (동의 받은 것)
  - 최소 5개, 가능하면 10개+
  - 각 후기: 학년 / 거주지 (동 단위) / 자녀와의 관계 / 후기 본문 / 작성 시기
  - 길이: 2~4문장 권장
  - **실제 이름은 노출하지 않고 이니셜 처리** (개인정보 보호)
- [ ] **졸업생 합격 사례** (선택)
  - 학교명만 (개인정보 X)
  - 예: "○○년 외고 △△외국어고 합격"
- [ ] **수강 평점 / 만족도 설문 결과** (보유 시)

## 4. 지도 / 위치 정보

- [ ] **카카오맵 또는 네이버맵 임베드 URL**
  - 학원 공식 채널에 등록되어 있어야 검색 가능
- [ ] **주차 안내**
  - 건물 자체 주차장 여부
  - 무료/유료 / 시간 제한
- [ ] **대중교통**
  - 가장 가까운 지하철역 + 출구 + 도보 분
  - 인근 버스 정류장 + 노선 번호

## 5. 상담 신청 폼 처리 방식

상담 폼은 **백엔드 없이** 어떻게 처리할지 결정 필요:

- [ ] **선택지 A — `mailto:`** (가장 간단)
  - 사용자가 폼 작성 → "신청" 클릭 시 이메일 클라이언트 자동 열림
  - 단점: 모바일에서 어색, 일부 사용자 메일 클라이언트 미설정
- [ ] **선택지 B — Google Form 임베드** (권장)
  - 학원 측이 Google Form 만들기 → iframe 임베드
  - 장점: 무료, 응답 자동 수집, 스프레드시트 연동
- [ ] **선택지 C — Naver 폼**
  - 한국 사용자에게 친숙
  - 네이버 카페/블로그 운영 시 자연스러움
- [ ] **선택지 D — 외부 SaaS (Tally, Typeform 등)**
  - UI는 가장 예쁘지만 무료 한도 있음
- [ ] **선택지 E — 별도 백엔드 / DB**
  - 2차 단계 (정적 사이트 → 동적 사이트 전환 시 고려)

> **1차 권장**: B (Google Form). 0원이고 학원 측 운영이 가장 편함.

## 6. 도메인 / 호스팅

- [ ] **도메인 보유 여부**
  - 보유 시: 현재 도메인 (예: minjok-eng.com)
  - 미보유 시: 신규 등록 필요
- [ ] **호스팅 환경**
  - 옵션 1: GitHub Pages (무료, 정적 사이트에 적합)
  - 옵션 2: Netlify (무료 + CDN + 폼 처리까지 가능)
  - 옵션 3: Cafe24 / 가비아 (국내 호스팅, 익숙함)
  - 옵션 4: 학원 자체 서버

## 7. 기타 결정 사항

- [ ] **개인정보처리방침** 문구 — 변호사/세무사 검토 받은 표준 양식
- [ ] **개인정보 수집·이용 동의서** 문구 (상담 폼용)
- [ ] **저작권 / 광고 문구 책임** — 사이트 푸터 표기
- [ ] **사이트맵.xml / robots.txt** — SEO를 위해 작성 (2차 단계)
- [ ] **구글 애널리틱스 / 네이버 서치어드바이저** 등록 여부

## 8. 진행 체크리스트

기획안 컨펌 → HTML 구현 흐름에서, 아래 순서로 자산을 받으면 효율적.

### Phase 1 (메인 페이지 구현 전)
- Hero 사진 1장
- 학원 기본 정보 (이름, 주소, 전화, 운영시간)
- 로고 (또는 텍스트 로고로 진행 결정)
- 상담 폼 처리 방식 (Google Form 등)

### Phase 2 (서브 페이지 구현 전)
- 원장 인사말 + 사진
- 강사진 정보 + 사진
- 6층 유치관 시설 사진 + 특별 활동 사진
- 학부모 후기 (5개 이상)

### Phase 3 (오픈 직전)
- 정확한 연혁
- 진학 실적
- 모든 프로그램 상세 (시간표, 교재 등)
- 도메인 / 호스팅 결정
- 개인정보 정책 문구

## 9. 더미값 임시 사용 일람

1차 HTML 구현 시 다음과 같이 더미값을 사용하고 식별 가능하도록 표기:

```
- 전화번호:  02-XXX-XXXX
- 주소:      서울 서대문구 ○○로 ○○
- 이메일:    info@minjok.kr
- 카카오톡:  @minjok-eng
- 원장 이름: ○○○
- 사진:      Unsplash 무료 이미지 (`unsplash.com/photos/...`)
- 후기 본문: 학원 분위기에 맞는 가공된 더미 (실제 후기 아님 명시)
- 연혁 숫자: 졸업생 ○○○명 등 ○ 표기
```

→ 실제 자산이 확보되는 대로 검색·교체. 더미값을 `data-dummy="true"` 같은 속성으로 마킹해두면 일괄 검색이 쉬워진다.
